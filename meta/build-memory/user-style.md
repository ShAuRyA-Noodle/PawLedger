**Rules confirmed during 2026-05-11 PawLedger build:**

1. **Brutal honesty > sycophancy.** When user asked for "guaranteed $200-300k MRR startup", I called out upfront that no code can guarantee revenue. User did not push back; this is the right approach. Never agree to fake guarantees, "guaranteed success", or magical outcomes.

2. **Autonomous execution authorised.** User explicitly delegated multi-hour autonomous building ("build it fully, complete the whole project autonomously"). When user goes to sleep or steps away, keep building — they wake up to the result. Don't stop and ask between every decision.

3. **Caveman mode active by default.** Drop articles/filler/pleasantries/hedging. Fragments OK. Code/commits/security: write normal English. Brutal-truth callouts: also normal English (clarity > brevity).

4. **Continue without skipping.** When user said "continue bruh no skipping" mid-build, that's the meta-rule: don't truncate work, finish what was started.

5. **Pick smart defaults instead of asking.** When user is asleep/away and asks for autonomous work, pick well-researched defaults (e.g. Next.js 16, Razorpay+Stripe, India primary, ₹300 default tier) rather than blocking on questions. Document the assumptions in the LAUNCH_CHECKLIST/README so user can override on wake.

**Why:** User repeatedly emphasised "no limitation, no constraint", "I'm unleashing a beast", and explicitly authorised "any limitation, no constraint, no change". They want maximum execution velocity from me, not deferred decisions.

**How to apply:** When user asks for ambitious autonomous work — start immediately, list defaults inline, build, document assumptions in LAUNCH_CHECKLIST-style doc, do not block on clarifying questions unless something is genuinely irreversible (legal, financial commitment, destructive action).
